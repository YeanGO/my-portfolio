01｜360 度績效互評系統

課程：Python 程式設計

以企業績效評核流程為發想，實作多角色互動的資訊系統。

專案簡介

360 度績效互評系統是一套以多來源評核概念為核心的網頁系統，模擬企業或組織中由不同角色共同參與評價與分析的流程。系統整合了自我評量、同儕互評、教師評分與管理者分析等功能，並透過資料彙整與權重計算，產出可供檢視的綜合評核結果。

本專案使用 Python FastAPI 建立後端服務，搭配前端網頁介面與資料庫設計，完成從使用者登入、評分資料送出、後端儲存、綜合分數計算，到後台分析呈現的完整流程。

專案發想與設計動機

在過去企業管理與人力資源工作的經驗中，我經常接觸績效評核、員工發展與跨角色意見整合等議題。
因此在 Python 程式設計課程中，我選擇以「360 度績效互評」作為系統開發主題，嘗試將熟悉的企業流程轉化為可操作的資訊系統。

將多角色評核流程數位化，呈現從資料輸入到結果分析的完整系統邏輯。
練習前後端資料串接、API 設計與資料庫儲存。
結合既有流程理解與資訊技術，建立具有實務情境的系統作品。

系統角色與流程設計

系統依據評核流程設計不同角色與操作權限，模擬組織中多方共同參與評估的情境。

學生 / 受評者

進行自我評量，並完成指定對象的同儕互評。

教師 / 評核者

針對學生進行評分。

管理者

檢視整體評核結果、加權分數與分析資訊。

  1. 使用者登入系統後，依身份進入不同操作頁面。
  2. 學生完成自我評量與同儕互評。
  3. 教師完成指定對象的評核。
  4. 系統彙整不同來源的評分資料。
  5. 依設定權重計算綜合結果。
  6. 管理者於後台檢視統計結果與分析資訊。

核心功能

多角色登入與權限區分

系統根據登入身份導向不同頁面，使學生、教師與管理者各自執行對應功能。

自評、同儕互評與教師評核

設計不同評核來源，模擬 360 度評核中多方回饋的資料蒐集方式。

六項能力指標評分

系統以六個構面進行評價，可對應到工作態度、工作績效、紀律規範、專業能力、學習成長與溝通協作等面向，建立更具結構性的評核資料。

綜合權重計算

系統將自評、同儕評核與教師評分依設定比例進行加權計算，產出綜合評核結果。

後台總覽與分析

管理者可檢視整體評核資料、分數彙整與視覺化結果，協助快速掌握個別受評者的整體表現。

評核結果延伸分析

系統進一步將評核分數轉化為多維度特徵，並進行相似性分析與結果呈現，提升資料應用層次。

技術實作

Python FastAPI HTML CSS JavaScript SQLite RESTful API Git GitHub

1. 前後端資料串接

前端頁面透過 API 與後端 FastAPI 服務互動，完成登入驗證、評分資料取得與評分結果送出。

2. 評分資料結構設計

針對自評、同儕互評與教師評分建立對應資料儲存方式，使系統能正確彙整不同來源的分數。

3. 評核結果聚合邏輯

後端負責整合各類評分資料，並依照設定權重計算最終分數，建立可供後台分析使用的結果資料。

4. 多角色頁面流程

依據不同使用者身份設計專屬頁面，呈現完整的業務流程與系統導覽邏輯。
